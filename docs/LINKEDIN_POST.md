## Primary — founder-led

I've been heads-down building **AuthorChain** — and the core just came together.

Independent authors deserve to *own* their work, not rent it from a platform.

So AuthorChain lets an author:
→ upload a manuscript (kept private)
→ register its SHA-256 hash on-chain as proof of authorship
→ sell it with Stripe, with royalties tracked automatically
→ deliver it securely to buyers (and only buyers)

Validated end to end today — **Publish → Prove → Sell → Earn → Deliver** — on Base
(testnet for now). The best part: the hash registered on-chain matches the exact
file the reader downloads, byte for byte. So an author can prove *"this manuscript
existed under my claim at this timestamp,"* verifiable by anyone on a public
blockchain.

Early, but real. Building this to give independent authors what they've never
really had: verifiable ownership, transparent royalties, and a direct line to
their readers.

**Publish. Own. Earn. Grow.**

## Alternate — technical / proof-forward

Most publishing platforms can prove a book *listing* exists.

AuthorChain proves the **actual manuscript file** exists — and that it was yours,
at a specific moment in time.

I just validated the full loop for **AuthorChain**, an AI-assisted Web3
publishing platform for independent authors:

📝 Author uploads a manuscript (stored privately)
🔒 We hash the real file (SHA-256) — the file never goes on-chain
⛓️ The hash is registered on Base Sepolia as proof of authorship
💳 Readers buy with Stripe; royalties are tracked automatically
📚 Buyers get secure access — non-buyers are blocked

The part I'm most proud of: the hash registered on-chain **matches the exact file
the buyer downloads**, byte for byte. An author can prove *"this manuscript
existed under my claim at this timestamp"* — verifiable by anyone on-chain.

It's an MVP (testnet + Stripe test mode today), but the whole loop works end to
end: **Publish → Prove → Sell → Earn → Deliver.**
